# Integration UI Storybook

인티그레이션 Frontend 팀에서 사용하는 공용 컴포넌트들을 참고 & 테스트하기 위한 Storybook 페이지입니다.



<div class="aside">
💡 <strong>NOTE</strong>: <code>🐞</code> 이모티콘을 붙여 버그가 있거나 개선이 필요함을 나타냅니다.
</div>

<div className="subheading">rules</div>

storybook을 작성할 때 아래와 같은 규칙들을 따릅니다.

- `.storybook/stories/[componentName].stories.js`에 story를 작성합니다.

- 기본 story를 만듭니다. ( Playground로 명명합니다. )

- Prop을 각각 story로 만듭니다.

- Prop story 이름 앞에는 colon(`:`)을 붙입니다. `예시) :Size`

- 필요에 따라 임의의 story를 만듭니다. 이 때는 colon(`:`)을 붙이지 않습니다. `예시) First Page`

- 버그가 있거나 개선이 필요한 컴포넌트 story의 경우 이름 뒤에 `Lady Beetle` 이모티콘을 붙입니다. `예시) :Color 🐞`
